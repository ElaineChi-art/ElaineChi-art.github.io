從零建置一套能跑深度學習的開發環境：Python／GPU／Colab 三種路線怎麼選

技能實驗室・每日課程 2026-07-22 約 6 分鐘 對應真專案主題：新手村｜系統操作講解
⚡ 30秒白話版

要跑深度學習，環境有三條路：本機純 Python 最單純但沒 GPU 就慢、本機裝 GPU 最快最自由但安裝最痛、雲端 Colab 零安裝馬上有免費 GPU 但會斷線。新手先用 Colab 把流程跑通，等到要反覆訓練、或資料不能外流，再考慮本機 GPU。選哪條的關鍵是「你現在卡在哪」，不是「哪個最強」。

SLIDE 1 簡報講解

這張圖在講的是，要跑深度學習，開發環境其實只有三條路，差別只在那三層——Python、框架、還有運算資源——到底裝在哪裡。路線一是本機純 Python，最單純，適合先把語法和流程跑通，但沒有 GPU，大模型會慢到不能用。路線二是本機加上 GPU，速度最快、資料也不外流，代價是安裝最痛，CUDA、驅動、框架的版本都要對得起來。路線三是雲端 Colab，開個瀏覽器就有免費 GPU，零安裝，缺點是連線會斷、免費額度有限。所以我的建議很簡單：新手先用 Colab 把東西跑起來，等到要反覆訓練、或資料不能外流，再回頭處理本機 GPU。選的關鍵永遠是你現在卡在哪，不是哪個聽起來最強。

壹、這是什麼

所謂「開發環境」，就是一套能讓深度學習程式真的跑起來的軟硬體組合。它至少疊了三層：Python 直譯器（實際執行程式碼的引擎）、深度學習框架（PyTorch 或 TensorFlow，把底層數學運算包成好用的函式）、以及最底下的運算資源（CPU 或 GPU）。三條路線的差別，其實只差在「這三層裝在哪裡、由誰負責」。本機路線是三層都在自己電腦；Colab 路線是三層都在 Google 的雲端主機，你只透過瀏覽器下指令。搞懂這件事，你就不會把「選環境」想得太玄——你只是在決定這三層放哪。

貳、為什麼業界在乎

環境選錯，會讓你連「開始學」都卡住。深度學習跟一般寫程式最大的不同，是它非常吃 GPU（一種原本用來畫圖形、很擅長同時做大量平行運算的晶片）。同一個模型，用 CPU 訓練可能要好幾小時，換 GPU 可能幾分鐘就跑完——這是量級上的差距，不是快一點而已。104 人力銀行《AI 人才趨勢報告》也強調，企業期待求職者能實際操作 PyTorch 這類工具、打通完整的應用情境，而不是只看過投影片。你得先有一個能反覆實驗的環境，才談得上累積這種手感。環境是所有後續學習的地基，地基歪了，後面每一步都在跟工具打架，而不是在學真正的東西。

參、實務怎麼用

放到真實情境會更具體。假設我進到玉山或 91APP 的資料團隊，被指派「先把這個模型在本地重現一次」。我不會一開始就在自己筆電硬裝 CUDA，因為那最容易卡上三天還跑不動。我會先在 Colab 把程式跑通、確認邏輯對，這是最快拿到「它能動」訊號的方式。等到要正式反覆訓練、或資料涉及客戶個資不能上傳雲端，我才切到公司配的 GPU 機器；這時安裝痛歸痛，但目標明確、而且身邊有人可以問。對我這種法律金融跨域背景的人來說，這個判斷順序本身就是能拿出來講的東西：先用低成本方式驗證可行性，再投入昂貴資源——這跟做法遵或風險評估的思路是同一套。

肆、常見誤區或追問延伸

最常見的誤區，是以為「裝了 GPU 版的框架，就等於程式在用 GPU」。實際上，PyTorch 裝錯版本、或 CUDA 跟顯卡驅動對不上時，程式往往會安靜地退回用 CPU 跑——它不會報錯，你只會覺得「怎麼這麼慢」。所以裝完環境第一件事，永遠是印出 torch.cuda.is_available() 確認 GPU 真的被抓到（回傳 True 才算數）。另一個容易被追問的點是 Colab 的免費額度：它不是無限的，長時間閒置或跑太久會被斷線回收，所以重要的訓練一定要把模型存到 Google Drive，不然辛苦幾小時、一斷全沒。

選環境不是選最強的那個，是選「能讓你今天就開始動手、明天不會被工具卡死」的那個。

🎤 面試官可能怎麼問
  1. 「你平常都怎麼建深度學習的開發環境？」 👉 這題其實在問：你是真的動手做過，還是只看過教學？ 她可以怎麼答：我會看情境。快速驗證想法或學新東西，我直接用 Colab，零安裝又有免費 GPU；要反覆訓練、或資料不能外流，才在本機裝對應版本的 PyTorch 配 GPU。我習慣裝完先跑 torch.cuda.is_available() 確認 GPU 有被抓到，再開始正式做。
  2. 「為什麼深度學習要用 GPU？CPU 不行嗎？」 👉 這題其實在問：你懂不懂底層為什麼快，還是只會照抄指令。 她可以怎麼答：CPU 也能跑，只是慢。神經網路的核心是大量矩陣運算，這些運算彼此獨立、可以同時做；GPU 剛好有成千上萬個核心，就是擅長這種平行運算。所以同一個模型，GPU 常常比 CPU 快一個數量級以上。
  3. 「Colab 跟本機環境，你會怎麼選？」 👉 這題其實在問：你會不會權衡取捨，而不是背一個標準答案。 她可以怎麼答：我的順序是先用 Colab 把流程跑通、確認邏輯對，因為它最快拿到「能動」的訊號、又不用裝環境。等到要正式訓練、或資料涉及個資不能上雲，再切到本機 GPU。原則是先驗證可行性，再投入成本。
  4. 「環境裝好之後，你怎麼確認 GPU 真的有在用？」 👉 這題其實在問：你知不知道「裝了不等於有用到」這個坑。 她可以怎麼答：我會先在 Python 裡跑 torch.cuda.is_available()，回傳 True 才代表框架有抓到 GPU；也可以用 nvidia-smi 看訓練時卡有沒有在動。因為版本對不上時程式會安靜地退回 CPU、不報錯，只是很慢，所以這個確認我一定會做。
  5. 「你自己在本機從頭裝過 CUDA 環境嗎？」 👉 這題其實在問：你的實戰深度到哪，坦白比唬爛加分。 她可以怎麼答（誠實的答法）：老實說我目前多半用 Colab，本機從零裝 CUDA 配驅動的完整經驗還不多。但我清楚主要的坑在版本對應——CUDA、顯卡驅動、框架三者要相容，我會照官方對應表一步步裝，裝完用 torch.cuda.is_available() 跟 nvidia-smi 驗證，遇到問題再從版本相容性查起。
