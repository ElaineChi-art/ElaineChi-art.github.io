參考文獻 · 技術與法律依據

本模型每一步驟均依下列權威技術規範與法律文獻製作，非泛論示意。

技術規範

  1. Wood, G. Ethereum: A Secure Decentralised Generalised Transaction Ledger（Yellow Paper）.
  2. ethereum.org Developer Docs — EVM · Transactions · Gas · Gasper.
  3. Buterin, V. & Griffith, V.（2020）Combining GHOST and Casper, arXiv:2003.03052.
  4. EIP-1559 — Fee Market Change（base fee + priority fee）.
  5. EIP-1967 — Standard Proxy Storage Slots.
  6. EIP-1822 — Universal Upgradeable Proxy Standard（UUPS）.
  7. OpenZeppelin Contracts Docs — Proxy · AccessControl · Pausable · ReentrancyGuard.
  8. Safe（前 Gnosis Safe）Documentation — Multisig（M-of-N、approveHash / execTransaction）.
  9. ERC-721 — Non-Fungible Token Standard（tokenURI）；ERC-2981 — NFT Royalty Standard（royaltyInfo）.
  10. IPFS（內容定址 CID）／ Arweave（permaweb）去中心化儲存規範.
  11. 實證平台：Royal.io、Opulous、Sound.xyz、Audius（音樂 NFT 版稅／收益分潤模式）.

法律文獻

  1. SEC v. Impact Theory, LLC（2023）— 首件 NFT 認定為未登記證券之執法案；和解要求修改程式碼以停止收取二級市場版稅。
  2. Regulation (EU) 2023/1114（MiCA）Recital 11；ESMA Guidelines — 「實質重於形式」、大量系列發行視為可替代之指標。
  3. CFTC v. Ooki DAO（2022）— 去中心化自治組織（DAO）無法人格仍可能被視為可歸責主體。
  4. 我國《洗錢防制法》、金管會 VASP 指導原則；MiCA 之 CASP 定義（VASP / SCSP 界分）。
  5. 我國《電子簽章法》§2、§6；民法（要約承諾、錯誤、詐欺、不當得利）；著作權法 §37。
  6. 姬政妤，《智慧合約之法制整合與智慧財產權保障研究——以虛擬資產服務法與音樂NFT為中心》（碩士論文）.
